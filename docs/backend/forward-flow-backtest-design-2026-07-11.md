# 순방향 호가 흐름 백테스트 설계

## 문제와 목표

현재 봉인 검증에서 OHLCV, 과거 테이커 흐름, 미결제약정, 프리미엄, 펀딩만으로는
목표인 일복리 `0.8%`, MTM MDD `40% 이하`, 청산 `0`, 의미 있는 거래 커버리지를
동시에 충족한 후보가 없다. 이후 Bybit 공식 historical-data portal에서 BTCUSDT
contract order-book snapshot/delta archive가 `2023-01-19` 이후 제공되는 것을
확인했다. 이 원본을 검증 없이 추정·합성하는 것은 허용하지 않으며, 원본 ZIP 해시와
완결 M1 커버리지를 남기는 import만 허용한다. 현재 확인된 `2025-08-21` 원본은
1,306분으로 불완전하므로 연속 관측 구간으로 취급하지 않는다.

이 작업의 목표는 **실제 순방향 수집 또는 공식 원본 아카이브에서 검증된 M1
호가/테이커 데이터만** 이용해 기존 인과 백테스트 엔진에서 재현 가능한 연구를
수행하는 것이다. 이 구현은 성과 주장이 아니며, 실거래 전략을 활성화하거나 변경하지
않는다.

## 범위와 비범위

포함:

- `OrderBookImbalanceBar`와 `TakerFlowBar`의 완료된 M1 관측치를 M5 버킷으로 묶는다.
- M5 설정 봉이 닫힌 시각까지 사용 가능한 관측치만 읽는다.
- 호가 방향 불균형 최소값과 평균 스프레드 최대값을 연구용 필터로 제공한다.
- 기존의 다음 M1 체결, 수수료, 슬리피지, 수량 제약, 레버리지, 청산, MTM MDD 계산을
  그대로 사용한다.
- 누락, 중복, 미완결 M1 관측치는 진입을 거절하고 사유를 보고한다.

제외:

- 새 실거래 전략 모드, 자동 루프, 기본 공격형 프로파일, 배포 환경변수 변경.
- 과거 호가 이력의 합성/추정, 봉인 구간을 이용한 파라미터 탐색, 성과 보장.
- 강제청산 흐름을 진입 필터로 사용하는 것. 초기에 관측 빈도가 낮을 수 있으므로
  지표 보존과 별도 진단까지만 허용한다.

## 사용 사례와 수용 기준

### 연구자가 수집 이후 구간을 재현한다

연구자는 인증된 기존 `/backtests/volume-flow/composite/run` API에 호가 필터가 있는
복합 레그와 `replayStartAt`/`replayEndAt`을 명시해 수집된 기간만 재현한다. 이 경로는
봉인 평가기가 이미 호출하는 연구 전용 경로다.

수용 기준:

- 호가 필터가 켜진 경우 M5 설정 시간대만 허용한다.
- 각 필요한 M5 버킷에는 정확히 연속된 다섯 개의 M1 호가 막대가 있어야 한다.
- 모든 M1 막대의 `availableAt`이 설정 봉 종가보다 늦으면 해당 설정은 거절된다.
- 보고서의 거래별 흐름 지표에는 방향 호가 불균형과 평균 스프레드가 포함된다.
- 복합 보고서는 필요한 호가/테이커 M5 버킷 수, 완결 버킷 수, 완결률을 함께 반환한다.

### 운영자가 누락 데이터를 발견한다

수용 기준:

- 저장소가 없으면 요청을 즉시 실패시킨다.
- 데이터가 비었거나 중복/누락이면 `MISSING_ORDER_BOOK_FLOW`를 집계한다.
- 스프레드 상한을 넘으면 `ORDER_BOOK_SPREAD_TOO_WIDE`를 집계한다.
- 방향성이 부족하면 `DIRECTIONAL_ORDER_BOOK_IMBALANCE_LOW`를 집계한다.

## 도메인과 계층

`ForwardMarketCaptureStore`는 수집 계층의 완료된 M1 호가 막대를 제공한다.
`VolumeFlowBacktestService`는 페이지 단위로 읽고 M5 버킷을 구성한다.
`VolumeFlowBacktestFlowContext`는 시간 인과성, 연속성, 방향/스프레드 조건을
판정한다. 라우트는 요청 열거형과 숫자 범위를 검증하고 서비스에 전달할 뿐 전략
판정을 수행하지 않는다.

`VolumeFlowCompositeBacktestService`는 같은 흐름 컨텍스트를 모든 레그에 한 번만
읽어 기간 지정 재현에 전달한다. 호가 필터가 있는 복합 보고서에는
`forwardCaptureReplayCoverage`를 붙인다. 봉인 평가기는 이 값이 필요한 경우 `100%`
완결이 아니면 `FORWARD_CAPTURE_COVERAGE_INSUFFICIENT`로 실패시킨다.

읽기 전용 백테스트이므로 트랜잭션이나 스키마 변경은 없다. 기존 SQLite 조회 한도
`10,000`개를 넘는 수집 구간은 열린 시각을 기준으로 페이지를 이어서 읽는다.

## 시간 인과성 계약

1. M1 호가 막대 `openedAt`은 `[t, t+1m)` 구간을 대표하고 `availableAt=t+1m`이다.
2. M5 버킷은 `[t, t+5m)`의 정확히 다섯 M1 막대로만 구성되며, `availableAt=t+5m`이다.
3. M5 설정 봉의 판단 시각은 `setupClosedAt=t+5m`이다. 이후 M1 호가 막대는 읽지 않는다.
4. 진입 및 청산은 기존 `causal-next-m1-open-v2` 체결 모델로만 계산한다.

## 구성 계약

- `minDirectionalOrderBookImbalance`: 방향 보정 뒤 필요한 최소 호가 불균형. `0..1`.
- `orderBookImbalanceDirectionMode`: `ALIGN_WITH_SIDE` 또는 `OPPOSE_SIDE`.
- `maxMeanOrderBookSpreadBps`: 같은 관측 창의 표본 가중 평균 스프레드 상한. `0 이상`.
- 호가 필터 중 하나라도 설정되면 `flowLookbackM1Candles`는 5의 배수여야 한다.

기본값은 모두 비활성이다. 즉 기존 백테스트, 현재 API 호출, 런타임 실행에는 행동
변화가 없다.

## 봉인 평가 연결

복합 레그 요청은 단일 백테스트와 같은 호가 필터 세 값을 지원한다. 따라서 미래
후보는 고정 JSON 전략 파일에 해당 값을 명시하고, 기존 봉인 평가 스크립트로
`/backtests/volume-flow/composite/run`을 호출한다. 호가 필터가 없는 과거 후보는
기존과 같은 캔들 커버리지 규칙만 적용된다.

호가 필터가 있는 후보는 캔들 커버리지와 별도로 요청된 M5 구간의 모든 필수
호가/테이커 버킷이 완결되어야 한다. 이 조건은 거래 수가 부족한 결과와 구분해
수집 부족 자체를 실패로 기록한다. 아직 실제 수집 기간보다 긴 1~60개월 구간을
생성하거나 기존 역사 구간에 호가 값을 합성하는 행위는 허용하지 않는다.

순방향 관측 구간이 최소 60개월 누적된 뒤에는
`scripts/forward-capture-sealed-protocol.mjs`가 명시적으로 지정된 월 경계 구간을
검사한다. M1 호가·테이커와 M1/M5/M15 캔들이 빠짐없이 존재할 때만, 고정 seed로
1~60개월 길이의 40개 구간과 SHA-256 해시를 생성한다. 한 분이라도 누락되면
프로토콜 파일을 만들지 않는다.

```bash
node scripts/forward-capture-sealed-protocol.mjs \
  --db=/path/to/bybit-trader.sqlite \
  --symbol=BTCUSDT \
  --start=2026-08-01T00:00:00.000Z \
  --end=2031-08-01T00:00:00.000Z \
  --seed=20310801 \
  --out=config/forward-capture-sealed-windows-v1.json
```

공식 아카이브는 현재 2023년 이후만 제공하고 `2025-08-21`에 확인된 원천 결측도
있으므로 1-60개월 목표나 41개월 연속 아카이브 프로토콜을 충족하지 못한다. 아카이브
연구는 실제로 검증된 연속 월 수를 넘지 않는 명시적 `--max-months`와 별도 source
provenance를 사용해야 하며, 이를 60개월 봉인 검증이나 실거래 승격으로 표현하면 안
된다. `bybit-official-orderbook-archive-and-taker-history` source를 선택하면
프로토콜 생성 시 모든 UTC 일자에 `historicalOrderBookImports`의 1,440분 manifest와
SHA-256이 있어야 한다. 따라서 누락 원천을 캔들 또는 수동 호가 행으로 채워도
프로토콜을 생성할 수 없다.

## 실패 처리와 검증

- 저장소 부재는 `IllegalArgumentException`으로 중단한다.
- 불완전하거나 중복된 M1 묶음은 보수적으로 진입을 거절한다.
- 최근/미래 호가 막대가 의도치 않게 포함되지 않는지 정확 조회 전용 테스트로 검증한다.
- 방향 통과, 방향 거절, 스프레드 거절, 누락 거절, 기존 체결/청산 회귀 테스트를 둔다.

## 승격 게이트

이 기능이 존재해도 성과는 미검증이다. 수집된 순방향 기간이 충분해진 후 사전에
고정한 후보를 별도 개발 구간에서 정의하고, 이후 새 랜덤 20~40개 봉인 구간에서
CDR `>=0.8%`, MTM MDD `<=40%`, 청산 `0`, 거래 및 활성일 커버리지를 **모두**
통과할 때만 별도 사람 승인 하에 승격을 검토한다.
