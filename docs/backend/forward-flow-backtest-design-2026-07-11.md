# 순방향 호가 흐름 백테스트 설계

## 문제와 목표

현재 봉인 검증에서 OHLCV, 과거 테이커 흐름, 미결제약정, 프리미엄, 펀딩만으로는
목표인 일복리 `0.8%`, MTM MDD `40% 이하`, 청산 `0`, 의미 있는 거래 커버리지를
동시에 충족한 후보가 없다. 과거 시점의 호가 잔량 이력은 보유하지 않았으므로, 이를
과거 구간으로 소급해 채우거나 기존 결과를 다시 튜닝하는 것은 허용하지 않는다.

이 작업의 목표는 **수집이 시작된 뒤의 실제 M1 호가/테이커 데이터만** 이용해 기존
인과 백테스트 엔진에서 재현 가능한 연구를 수행하는 것이다. 이 구현은 성과 주장이
아니며, 실거래 전략을 활성화하거나 변경하지 않는다.

## 범위와 비범위

포함:

- `OrderBookImbalanceBar`와 `TakerFlowBar`의 완료된 M1 관측치를 M5 버킷으로 묶는다.
- M5 설정 봉이 닫힌 시각까지 사용 가능한 관측치만 읽는다.
- 호가 방향 불균형 최소값과 평균 스프레드 최대값을 연구용 필터로 제공한다.
- 기존의 다음 M1 체결, 수수료, 슬리피지, 수량 제약, 레버리지, 청산, MTM MDD 계산을
  그대로 사용한다.
- 누락, 중복, 미완결 M1 관측치는 진입을 거절하고 사유를 보고한다.

제외:

- 새 실거래 전략 모드, 자동 루프, 기본 공격형 프로파일, 배포 환경변수 변경.
- 과거 호가 이력의 합성/추정, 봉인 구간을 이용한 파라미터 탐색, 성과 보장.
- 강제청산 흐름을 진입 필터로 사용하는 것. 초기에 관측 빈도가 낮을 수 있으므로
  지표 보존과 별도 진단까지만 허용한다.

## 사용 사례와 수용 기준

### 연구자가 수집 이후 구간을 재현한다

연구자는 인증된 기존 `/backtests/volume-flow/run` API에 호가 필터 값을 명시해
수집된 기간만 재현한다.

수용 기준:

- 호가 필터가 켜진 경우 M5 설정 시간대만 허용한다.
- 각 필요한 M5 버킷에는 정확히 연속된 다섯 개의 M1 호가 막대가 있어야 한다.
- 모든 M1 막대의 `availableAt`이 설정 봉 종가보다 늦으면 해당 설정은 거절된다.
- 보고서의 거래별 흐름 지표에는 방향 호가 불균형과 평균 스프레드가 포함된다.

### 운영자가 누락 데이터를 발견한다

수용 기준:

- 저장소가 없으면 요청을 즉시 실패시킨다.
- 데이터가 비었거나 중복/누락이면 `MISSING_ORDER_BOOK_FLOW`를 집계한다.
- 스프레드 상한을 넘으면 `ORDER_BOOK_SPREAD_TOO_WIDE`를 집계한다.
- 방향성이 부족하면 `DIRECTIONAL_ORDER_BOOK_IMBALANCE_LOW`를 집계한다.

## 도메인과 계층

`ForwardMarketCaptureStore`는 수집 계층의 완료된 M1 호가 막대를 제공한다.
`VolumeFlowBacktestService`는 페이지 단위로 읽고 M5 버킷을 구성한다.
`VolumeFlowBacktestFlowContext`는 시간 인과성, 연속성, 방향/스프레드 조건을
판정한다. 라우트는 요청 열거형과 숫자 범위를 검증하고 서비스에 전달할 뿐 전략
판정을 수행하지 않는다.

읽기 전용 백테스트이므로 트랜잭션이나 스키마 변경은 없다. 기존 SQLite 조회 한도
`10,000`개를 넘는 수집 구간은 열린 시각을 기준으로 페이지를 이어서 읽는다.

## 시간 인과성 계약

1. M1 호가 막대 `openedAt`은 `[t, t+1m)` 구간을 대표하고 `availableAt=t+1m`이다.
2. M5 버킷은 `[t, t+5m)`의 정확히 다섯 M1 막대로만 구성되며, `availableAt=t+5m`이다.
3. M5 설정 봉의 판단 시각은 `setupClosedAt=t+5m`이다. 이후 M1 호가 막대는 읽지 않는다.
4. 진입 및 청산은 기존 `causal-next-m1-open-v2` 체결 모델로만 계산한다.

## 구성 계약

- `minDirectionalOrderBookImbalance`: 방향 보정 뒤 필요한 최소 호가 불균형. `0..1`.
- `orderBookImbalanceDirectionMode`: `ALIGN_WITH_SIDE` 또는 `OPPOSE_SIDE`.
- `maxMeanOrderBookSpreadBps`: 같은 관측 창의 표본 가중 평균 스프레드 상한. `0 이상`.
- 호가 필터 중 하나라도 설정되면 `flowLookbackM1Candles`는 5의 배수여야 한다.

기본값은 모두 비활성이다. 즉 기존 백테스트, 현재 API 호출, 런타임 실행에는 행동
변화가 없다.

## 실패 처리와 검증

- 저장소 부재는 `IllegalArgumentException`으로 중단한다.
- 불완전하거나 중복된 M1 묶음은 보수적으로 진입을 거절한다.
- 최근/미래 호가 막대가 의도치 않게 포함되지 않는지 정확 조회 전용 테스트로 검증한다.
- 방향 통과, 방향 거절, 스프레드 거절, 누락 거절, 기존 체결/청산 회귀 테스트를 둔다.

## 승격 게이트

이 기능이 존재해도 성과는 미검증이다. 수집된 순방향 기간이 충분해진 후 사전에
고정한 후보를 별도 개발 구간에서 정의하고, 이후 새 랜덤 20~40개 봉인 구간에서
CDR `>=0.8%`, MTM MDD `<=40%`, 청산 `0`, 거래 및 활성일 커버리지를 **모두**
통과할 때만 별도 사람 승인 하에 승격을 검토한다.
