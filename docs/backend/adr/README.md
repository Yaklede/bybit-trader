# Backend ADR

## 목표

`docs/backend/adr`은 장기 보존할 백엔드 아키텍처 의사결정 기록(ADR)을 보관한다.

## 적용 범위

- 기술 선택, 아키텍처 방향, 운영상 되돌리기 어려운 결정처럼 결정 배경과 결과를 남겨야 하는 경우에 적용한다.
- 현재 유효한 강제 규칙은 `docs/rules/backend`가 소유한다.

## 규칙

- BACKEND-ADR-001: ADR은 의사결정 기록이며 규칙 원문이 아니다.
- BACKEND-ADR-002: ADR의 결정이 반복 적용 규칙으로 굳어지면 사용자 확인 후 `docs/rules/backend`에 반영한다.
- BACKEND-ADR-003: ADR은 결정 배경, 선택지, 채택 결정, 영향 범위, 후속 조치를 포함한다.
